mold is a drop-in replacement for the system linker. Most build systems provide a way to pass extra linker flags or override the linker command. The sections below show how to wire mold into common build systems.

## CMake

Pass the linker flag via `CMAKE_EXE_LINKER_FLAGS` (and the equivalent variables for shared libraries and modules):

<Steps>
  <Step title="Configure with linker flags">
    ```shell
    cmake \
      -DCMAKE_EXE_LINKER_FLAGS="-fuse-ld=mold" \
      -DCMAKE_SHARED_LINKER_FLAGS="-fuse-ld=mold" \
      -DCMAKE_MODULE_LINKER_FLAGS="-fuse-ld=mold" \
      -B build
    ```
  </Step>
  <Step title="Build">
    ```shell
    cmake --build build -j$(nproc)
    ```
  </Step>
</Steps>

Alternatively, set the flags inside your `CMakeLists.txt`:

```cmake CMakeLists.txt
if(UNIX AND NOT APPLE)
  add_link_options(-fuse-ld=mold)
endif()
```

<Note>
`add_link_options` requires CMake 3.13 or later. For older versions, append to `CMAKE_EXE_LINKER_FLAGS` directly using `set(CMAKE_EXE_LINKER_FLAGS "${CMAKE_EXE_LINKER_FLAGS} -fuse-ld=mold")`.
</Note>

## Make

Set the `LDFLAGS` environment variable before invoking `make`:

```shell
make LDFLAGS="-fuse-ld=mold"
```

Or export it so it applies to all `make` invocations in the current shell:

```shell
export LDFLAGS="-fuse-ld=mold"
make
```

If modifying `LDFLAGS` is not possible, use `mold -run` to intercept linker calls instead. See the [mold -run guide](/guides/mold-run) for details.

## Ninja

Ninja itself does not manage linker flags — those come from the build description generated by CMake, Meson, or another meta-build tool. Follow the instructions for your meta-build tool.

For a hand-written `build.ninja`, add the flag to your link rule:

```ninja build.ninja
rule link
  command = clang -fuse-ld=mold $in -o $out
```

## Conan

Conan can download mold and configure it as the linker for your project and all its dependencies built from source. See the official Conan Center recipe for setup instructions:

## Nim

Create `config.nims` in your project directory:

```nim config.nims
when findExe("mold").len > 0 and defined(linux):
  switch("passL", "-fuse-ld=mold")
```

`mold` must be available on `PATH` for `findExe` to find it. This example uses GCC as the linker driver with the `-fuse-ld` option, which requires GCC 12.1 or later.

To apply the setting to all Nim projects on your machine, add the same snippet to `~/.config/config.nims`.

<Warning>
If your GCC is older than 12.1, `-fuse-ld=mold` will fail. In that case, use `-B/usr/libexec/mold` via `switch("passL", "-B/usr/libexec/mold")` instead.
</Warning>

## GitHub Actions

Use the [setup-mold](https://github.com/rui314/setup-mold) action to install mold on GitHub-hosted runners before your build step:

```yaml .github/workflows/build.yml
steps:
  - uses: actions/checkout@v4

  - name: Install mold
    uses: rui314/setup-mold@v1

  - name: Build
    run: cmake -DCMAKE_EXE_LINKER_FLAGS="-fuse-ld=mold" -B build && cmake --build build
```

<Tip>
GitHub Actions runners use 4-core machines. Even on a 4-core machine, mold is significantly faster than the default GNU linker for large programs.
</Tip>
