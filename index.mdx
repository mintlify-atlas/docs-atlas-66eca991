mold is a modern, massively parallel Unix linker designed to minimize build times — especially during rapid debug-edit-rebuild cycles. It is several times faster than the LLVM lld linker, which was itself a major improvement over GNU ld.

## Why mold?

Linking is the final step of every C, C++, or Rust build — and it can be surprisingly slow. mold addresses this with a fundamentally parallel architecture that keeps all CPU cores busy throughout the linking process.

| Program (output size)         | GNU ld  | GNU gold | LLVM lld | mold   |
|-------------------------------|---------|----------|----------|--------|
| MySQL 8.3 (0.47 GiB)          | 10.84s  | 7.47s    | 1.64s    | 0.46s  |
| Clang 19 (1.56 GiB)           | 42.07s  | 33.13s   | 5.20s    | 1.35s  |
| Chromium 124 (1.35 GiB)       | N/A     | 27.40s   | 6.10s    | 1.52s  |

## Key features

- **Massively parallel** — uses all CPU cores throughout the entire link process
- **Drop-in replacement** — fully compatible with GNU ld command-line interface
- **20+ architectures** — x86-64, i386, ARM64, ARM32, RISC-V, PowerPC, s390x, SPARC64, m68k, SH-4, LoongArch
- **Reproducible builds** — identical output for identical inputs, always
- **ICF** — Identical Code Folding merges duplicate functions to reduce output size
- **LTO support** — works with GCC and Clang link-time optimization
- **Separate debug files** — bundles debug info separately so executables launch faster
- **GDB index** — generates `.gdb_index` for faster debug sessions

<Steps>
  <Step title="Install mold">
    Install via your Linux distribution's package manager, or build from source with CMake.
  </Step>
  <Step title="Point your compiler at mold">
    Pass `-fuse-ld=mold` to Clang or GCC 12.1+. For older GCC, use `-B/usr/libexec/mold`.
  </Step>
  <Step title="Build your project">
    Run your normal build command. mold intercepts the linker invocation automatically.
  </Step>
  <Step title="Verify">
    Confirm mold was used by running `readelf -p .comment <your-binary>` and checking for the mold identifier.
  </Step>
</Steps>
