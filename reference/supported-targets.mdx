mold supports 20 CPU target architectures across ELF-based operating systems. All targets are included in every standard mold build.

## Architecture-specific features

Different targets have unique linking behaviors and optimizations. The most notable are described below.

### x86-64 (`X86_64`)

mold supports the `-z rewrite-endbr` option exclusively on x86-64. When enabled, mold performs a whole-program analysis to identify functions whose addresses are actually taken and rewrites unnecessary `ENDBR` (End Branch) landing pad instructions with no-ops. This reduces the attack surface for control-flow hijacking without changing program semantics.

x86-64 also supports Intel Control-flow Enforcement Technology (CET) reporting via `-z cet-report`, and IBT-enabled PLT generation via `-z ibt` and `-z ibtplt`.

### AArch64 (`ARM64LE`, `ARM64BE`)

AArch64 uses a 64 KiB default maximum page size (versus 4 KiB on x86-64, i386, and RISC-V). The `-z max-page-size` option can override this. AArch64 also supports the `--execute-only` option for execute-only memory pages on processors that implement the feature.

### ARM 32-bit (`ARM32LE`, `ARM32BE`)

The ARM 32-bit backend implements **thunks**: small trampolines inserted by the linker when a branch target is out of the direct branch range. This is necessary because ARM branch instructions encode a limited offset, and large programs may require jumps that exceed that range. mold inserts thunks automatically.

### RISC-V (`RV32LE`, `RV32BE`, `RV64LE`, `RV64BE`)

RISC-V supports **linker relaxation**. Certain instruction sequences generated by the compiler can be rewritten by the linker into shorter, more efficient forms once final symbol addresses are known. This is controlled by the `--relax` / `--no-relax` option (enabled by default). The default maximum page size for RISC-V is 4 KiB.

### PowerPC (`PPC32`, `PPC64V1`, `PPC64V2`)

mold supports three distinct PowerPC targets:

- **`PPC32`** — 32-bit big-endian PowerPC.
- **`PPC64V1`** — 64-bit big-endian PowerPC using the older ELFv1 ABI (used by classic Linux on POWER systems).
- **`PPC64V2`** — 64-bit little-endian PowerPC using the newer ELFv2 ABI (used by modern Linux on POWER8 and later).

### IBM s390x (`S390X`)

IBM's mainframe architecture, also known as z/Architecture. 64-bit big-endian.

### LoongArch (`LOONGARCH32`, `LOONGARCH64`)

Both 32-bit and 64-bit LoongArch targets are supported. LoongArch is a RISC ISA developed by Loongson Technology, used in Chinese-market processors. Both variants are little-endian.

## Building for a subset of targets

By default, mold is compiled with support for all 20 targets. You can restrict the build to a specific subset by passing the `MOLD_TARGETS` CMake option:

```bash
cmake -DMOLD_TARGETS='X86_64;ARM64LE' -B build
cmake --build build -j$(nproc)
```

<Warning>
Limiting `MOLD_TARGETS` is **strongly discouraged** for any binary you share with others or distribute as a package. mold's policy is that all published builds of the same version have the exact same feature set, which is essential for build reproducibility. Use this option only for personal iterative development where fast build times matter more than completeness.
</Warning>

## Selecting a target at link time

mold selects the output target from the input object files automatically. You can also specify a target explicitly using the `-m` flag, which accepts the standard GNU linker emulation names (for example, `elf_x86_64`, `aarch64linux`, `elf32_x86_64`).

```bash
mold -m elf_x86_64 -o output input.o
```
